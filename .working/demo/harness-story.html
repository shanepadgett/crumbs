The harness decides the bill

We pay for AI coding by the token. The model sets the rate, but the harness around it decides how many tokens we buy, and the harnesses our engineers use today were built by the companies selling those tokens. This note prices the difference between a default harness and one engineered to be token-conscious. Nothing here is a benchmark yet. The point is the shape of the math, and the shape holds at any plausible setting.

Every turn re-sends the whole conversation

A language model keeps no memory between steps. When an agent takes its fortieth turn, the harness uploads turns one through thirty-nine again, and the meter runs on all of it. The file it read at the start gets bought again. The test log from an hour ago gets bought again. Context grows a little every turn, so the total input bill grows roughly with the square of session length.

Output tokens get the attention because the per-token rate is five to six times higher. In agent loops the volume runs the other way: input tokens outnumber output tokens by enormous ratios, and nearly all of that input is repetition.

One afternoon, replayed twice

Here is a small, ordinary task: a checkout test is failing. Fix it, then start the follow-up the user asks for. Same model, same tools, same prompts on both sides. The default harness keeps every byte it has ever seen. The token-conscious harness writes full outputs to a local artifact store, hands the model a short capsule with a retrieval handle, and marks anything an edit invalidated as stale. Step through it.

By the time the user asks for the follow-up, the default harness re-sends about 87,000 tokens per turn and the conscious one about 5,100. Nothing was deleted. The full test logs, the search flood, and the original file snapshot all sit in the artifact store, one retrieval call away.

Waste has a shape

Four patterns cover most of what default harnesses re-send.

Stale reads. The agent reads a file, edits it, and the original copy stays in context. The cost is the small problem. The model can reason from the outdated copy and write a wrong edit on top of it. Marking old reads stale is a correctness fix that happens to save tokens.

Repeat reads. Agents re-read the same file three or four times in a session. A content hash turns the repeats into "unchanged since last read" plus the line ranges already shown.

Victory laps. A 4,200-line test log that passed carries one useful bit: it passed. The default harness re-sends the whole log on every later turn until the session ends.

Search floods. A code search returns 312 hits and three of them matter. Keep the three exact, store the rest, and hand the model a search handle for the stored set.

Edits bill twice

Our baseline agent edits files by quoting. One tool call per change, and each call spells out the old text and the new text in full. Three changes to one file means three round-trips, with the entire context re-uploaded between each one. So an edit costs twice: once for retyped code at output prices, and again for the extra turns of re-sent input.

A grounded patch removes both costs. The model proves it has the current file with a short content hash, then edits by line anchor: replace lines 42 through 42, delete 80 through 95, insert after 120. Several files, one call. If the file changed since the model last saw it, the hash mismatches and the patch is rejected before anything is written. The model gets told to re-read. That failure mode is strictly safer than string matching, which can silently hit the wrong occurrence.

Yes, this breaks the cache. Do it anyway.

The first objection from anyone who knows the billing model: rewriting old context invalidates the prompt cache. True. Providers bill cached input at a tenth of list price, and the discount only applies to a byte-identical prefix. Curating old context means paying full input price for one turn.

The rewrite pays for itself in four to five turns at current list prices, and agent sessions run far longer than five turns. A cache hit is a 90% discount on tokens that should not be in the prompt at all. Shrink first, cache what remains.

Full windows make agents worse

Money is the smaller half of the argument. Models reason measurably worse as their context fills; long-context degradation shows up in every published evaluation that looks for it. And when a default harness finally hits its window limit, it auto-compacts: a lossy summary replaces the history. The agent forgets which fixes it already tried, re-reads files it had open, re-runs suites that already passed, and sometimes re-applies a change the engineer reverted on purpose. The session pays for the same work twice while the engineer babysits it. This is the rework our people describe when they say the agent "got dumb" an hour in.

A harness that keeps working context at a quarter of the window never hits that wall. When compaction is genuinely needed, it writes a structured dossier: the goal, the user's constraints verbatim, what changed, what failed, and handles to every stored artifact. The next turn recovers detail instead of rediscovering it.

Two thousand engineers, priced

Apply the per-session math at our scale, on the models we actually buy. The defaults below are conservative for engineers who use agents all day: six sessions, forty turns each. The model is deliberately generous to the default harness. It gets a perfect cache hit on every single turn, which real traffic never achieves, and the rework bill from the previous section is left out entirely. Read the gap as a floor.

The per-engineer number explains something we already watch happen: at the default settings above, on Opus, the default harness costs $583 per engineer per month, which is how people burn through a $400 monthly budget in days. The answer so far has been rationing. A better answer is a harness that stops re-buying the same tokens.

What this is, and what it isn't

These are modeled costs on published list prices, with the assumptions printed under each figure. No benchmark numbers exist yet, and I won't pretend otherwise. The evaluation plan is written: stale-read traps, needle recovery out of capsuled artifacts, continuation quality after compaction, and a solve-rate regression suite. Savings that make the agent dumber don't count, so solve rate is the gating metric, measured before and after each mechanism ships.

The tradeoffs are engineered rather than wished away. Retrieval round-trips cost tokens, so the added tool surface stays at two tools. A capsule can hide a detail the model needed, so every capsule carries a retrieval handle and the exact bytes stay on disk. Curation breaks the cache, and the break-even math above prices that honestly.

One more thing worth saying plainly: this layer will not come from the model vendors. They bill per token, and their harnesses reflect it. Context engineering lives in the harness, the harness is software we can own, and whoever owns the harness owns the cost curve, the telemetry, and the leverage in the next vendor negotiation.

Pay the model to think. Stop paying it to re-read its own inbox.
