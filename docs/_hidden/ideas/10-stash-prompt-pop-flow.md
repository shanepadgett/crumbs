# Stash Prompt and Pop Flow

Let users stash the current prompt or drafting state and bring it back later without losing the exact wording they were building toward. This would be useful when a conversation gets interrupted by a side task, a context switch, or a command that temporarily changes the user's focus.

The "pop" step could restore the saved prompt directly or reopen it in an editable composer state before sending. That makes the feature useful both as a simple queue and as a way to preserve half-formed requests that need one more pass before execution.

The optional code-style chooser hint suggests this could become more than a raw text stash. It could be a small prompt workspace that stores text plus lightweight execution preferences, making it easier to resume not just what the user wanted to ask, but how they wanted Pi to respond.
